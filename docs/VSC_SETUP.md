# VS Code Setup

1. Open the repo root in VS Code.
2. Read the core docs before editing code:
   - `docs/ARCHITECTURE.md`
   - `docs/PACKAGE_CONTRACTS.md`
   - `docs/BUILD_ORDER.md`
3. Implement in package order.
4. Keep `out/` out of source control.
5. Do not wire app code against imaginary APIs.

## Suggested panels
- Explorer: repo tree
- Markdown preview: docs
- Terminal: FARD commands
- Problems: errors and missing pieces
